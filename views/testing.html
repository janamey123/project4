In order to find out if your model will work well with new cases you have to try it out with new cases.
But this should not be done by starting the production since a bad outcome could have a bad influence on your reputation.

Split your data into two sets

Training Set

Train your system with the training.

Test Set

Test your system with the test set.

How to split your data

Training Set

Use 80% of your data as training set.

Test Set

Use 20% of your data as test set. The fraction of the test set usually gets lower when the amount of data gets bigger.
“As one Google Translate engineer put it, "when you go from 10,000 training examples to 10 billion training examples, it all starts to work. Data trumps everything.”
- Garry Kasparov, Deep Thinking: Where Machine Intelligence Ends and Human Creativity Begins

Which model should you use?

Training Set

Train two or more models with your training set.

Test Set

Compare how well the models work with the test set afterwards. Therefore, you can compare the so called generalization error. The generalization error is the error rate on new cases.
